Я провел расследование ваших проблем и подготовил план решения.

### 1. Проблема: Несовпадение версий (9.0.0 vs 9.0.1)
**Причина:** Вы видите версию 9.0.1 в редакторе (VS Code), но при запуске Python загружает старую скомпилированную версию файла из папки `__pycache__`. Это частая ситуация при обновлении файлов без перезапуска среды.
**Решение:** Я удалю все папки `__pycache__`, чтобы заставить Python перекомпилировать код и подхватить актуальное значение `VERSION = "9.0.1"`.

### 2. Проблема: Ошибка `Failed to extract PyQt6\QtQuick.pyd` на другом ПК
**Причина:** 
1.  Эта ошибка означает, что в EXE-файл попал модуль `QtQuick`, который не используется в вашем проекте (я проверил код — импортов нет).
2.  Ошибка "decompression resulted in return code 0" часто вызвана тем, что антивирус на другом ПК блокирует распаковку временных файлов во время запуска, либо файл был собран с ошибками.
**Решение:** Скрипт сборки `build_exe_onefile.py` уже содержит настройки для исключения `QtQuick` (`--exclude-module=PyQt6.QtQuick`). Я запущу полную пересборку проекта. Новый EXE-файл будет:
*   Легче (без лишних библиотек).
*   Не будет пытаться распаковать `QtQuick.pyd`, что устранит ошибку.

### План действий:
1.  **Очистка:** Удалить все временные файлы Python (`__pycache__`) и старые сборки (`build`, `dist`).
2.  **Сборка:** Запустить `build_exe_onefile.py` для создания чистого релиза версии 9.0.1.
3.  **Проверка:** Вы сможете запустить новый `MoneyTracker.exe` из папки `Release_9.0.1` и убедиться, что версия отображается корректно и ошибка исчезла.
